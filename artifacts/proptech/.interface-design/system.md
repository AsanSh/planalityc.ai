# Design System — proptech (Asset-Manager)

Источник правды по UI-паттернам фронтенда. Аудит: `/audit <path>` сверяет код с этим файлом.

## Tables (ОБЯЗАТЕЛЬНО)

**Все новые таблицы — только через `<DataTable>`** (`src/components/data-table.tsx`).
Запрещено: сырой `<table>` и голый shadcn `<Table>` для отображения списков/данных.

Почему: `DataTable` даёт из коробки единый UX — сортировку, глобальный поиск,
плотность (compact/normal/comfortable), видимость/порядок/закрепление/ширину
колонок, CSV-экспорт, сохранение раскладки на пользователя (локально+сервер),
колонку действий `__actions`, sticky тёмную шапку и корректную тёмную тему.

Правила таблиц:
- Данные строятся через `columns: ColumnDef[]` + `<DataTable tableId=… columns=… data=… />`. `tableId` уникален (нужен для сохранения раскладки).
- Действия строки — только в колонке с id `__actions` (справа). Не рассыпать кнопки по ячейкам.
- Пустое состояние — через проп `emptyState`, не ad-hoc строкой в разметке.
- Выравнивание/формат — через `meta` колонки (`align`, `financeAmount`, `grow`, `truncate`, `pinned`), не инлайном.
- Шапка тёмная `bg-slate-950` + `text-white/72` + `text-[10px] uppercase tracking-[0.12em]` — задаётся компонентом, не переопределять на странице.
- `ui/table.tsx` (shadcn `Table`) — только для нетабличных списков/структурной разметки, не для датагридов.

Существующие 35 сырых `<table>` и 21 голый `<Table>` — легаси; мигрировать волнами
(rental → construction → crm → settings) на `DataTable`.

## Dark theme (ОБЯЗАТЕЛЬНО)

Тема «тёмный хром + светлый контент»: `.dark` на `<html>` делает тёмными сайдбар/шапку,
но контент-область остаётся светлой (`#F5F8FA`, инлайн в layout). При этом `.dark`
флипает семантические токены (`foreground`/`card-foreground`/`muted-foreground`) в светлые
→ светлый текст на светлой карточке/таблице = невидим.

Правила:
- НЕ пытаться делать карточки/таблицы тёмными в dark (базовый `.am-card` рисует светлый
  градиент без dark-override — не побороть; `dark:bg-card` бесполезен).
- Текст на светлых поверхностях в dark должен быть ТЁМНЫМ. Предпочтительно — пин токенов
  на корне блока: `dark:[--foreground:222_48%_10%] dark:[--card-foreground:222_48%_10%] dark:[--muted-foreground:215_16%_47%]`.
  Он чинит весь вложенный текст на утилитах `text-*-foreground` разом (таблицы и `DataTable`
  уже пинят токены на корне — там ничего не надо).
- Голый inherited-текст без утилиты цвета (напр. `<h1 className="text-2xl font-bold">` вне
  карточки) пин НЕ достаёт — ему нужен явный `dark:text-slate-900`.
- Запрещён хардкод `dark:text-slate-200`/светлых оттенков на светлом фоне — перебивает токены.
- Намеренно-тёмные карточки используют фиксированный `text-white` (не токены).

## Spacing

База 4px. Допустимая шкала (Tailwind): 4, 8, 12, 16, 20, 24, 32… Не использовать
значения вне сетки. Плотность таблиц: compact `py-1`, normal `py-2.5`, comfortable `py-4`.

## Depth

Мягкие тени через токены `--am-shadow*` (`am-card`, `am-shell-*`). Радиусы: контролы —
`var(--radius)`; карточки — 24px (`am-card`), 12–16px для вложенных блоков. Не добавлять
произвольные `box-shadow`/градиенты помимо `am-*`-паттернов.

## Color / tokens

Единый источник — токены `--am-*` в `src/index.css` (surface/border/text-strong/text/
text-muted/text-subtle) и семантические shadcn-токены. Бренд — cyan/teal (`am-brand`).
Не вводить произвольные hex; использовать токены или утилиты Tailwind из палитры slate/cyan/
emerald (статусы). Финансовые суммы — моно-шрифт (`am-card` уже включает tabular-nums).

## Toolbar над таблицей

Конвенция — `am-table-toolbar` (поиск слева, гибкий разделитель `flex-1`, справа плотность/
экспорт/столбцы). Ширины контролов — через flex/`minmax`, не хардкод-класс `.am-control.w-*`.
